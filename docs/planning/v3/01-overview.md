# V3 시험지 제작 파이프라인 — 설계 개요

## 배경

### V1 문제점
- PDF 전체를 텍스트로 변환 → 수식/그림/레이아웃 정보 손실 → **문제 왜곡**
- 20문제를 한 에이전트가 일괄 처리 → 뒤쪽 문제 품질 저하
- 해설 생성과 검증이 분리되지 않음 → 오류를 자체 감지 못함

### V2 개선 시도
- 문제별 이미지 슬롯(QuestionSlotGrid) 도입
- PDF + 개별 이미지 2차 검증
- 그러나 파이프라인 구조는 V1과 동일 (reader → solver → figure → builder → checker)

### V3 방향
- **이미지 기반**: 문제별 크롭 이미지를 직접 에이전트에 전달 (텍스트 변환 왜곡 제거)
- **문제별 병렬**: 추출/해설/검증을 문제 단위로 분리, 4개씩 병렬 처리
- **생성 ≠ 검증**: solver와 verifier를 완전 분리, 최대 3회 재시도
- **교과 순서 반영**: 단원분류표 컨텍스트를 해설 생성에 제공

## V3 파이프라인

```
┌─────────────────────────────────────────────────────────┐
│  [0] 프론트엔드 — 수동 크롭                                │
│      사용자가 문제별 이미지를 슬롯에 붙여넣기                   │
│      (자동 크롭은 파이프라인 안정화 후 마지막에 구현)            │
└───────────────────────┬─────────────────────────────────┘
                        ▼
┌─────────────────────────────────────────────────────────┐
│  [1] Extractor (문제별, 4개 병렬)                          │
│      입력: 문제 이미지 1장                                  │
│      출력: 문제 JSON (text, equation, choices, score)       │
│      에이전트: ngd-exam-extractor (신규)                    │
└───────────────────────┬─────────────────────────────────┘
                        ▼
┌─────────────────────────────────────────────────────────┐
│  [2] Solver (문제별, 4개 병렬)                             │
│      입력: extractor JSON + 단원분류표 + 해설 규칙            │
│      출력: explanation_parts JSON                          │
│      에이전트: ngd-exam-solver (기존 강화)                   │
└───────────────────────┬─────────────────────────────────┘
                        ▼
┌─────────────────────────────────────────────────────────┐
│  [3] Verifier (문제별, 4개 병렬)                           │
│      입력: 문제 이미지 + extractor JSON + solver JSON        │
│      검증: 수학적 정확성 / 교과 범위 / 서식 규칙               │
│      실패 시 → solver 재호출 (최대 3회)                      │
│      에이전트: ngd-exam-verifier (신규)                     │
└───────────────────────┬─────────────────────────────────┘
                        ▼
┌─────────────────────────────────────────────────────────┐
│  [4] Figure (그림 있는 문제만)                              │
│      입력: 문제 이미지에서 그림 영역 crop                     │
│      처리: nano-banana 재생성 → 트리밍 + 워터마크             │
│      에이전트: ngd-exam-figure (기존)                       │
└───────────────────────┬─────────────────────────────────┘
                        ▼
┌─────────────────────────────────────────────────────────┐
│  [5] Builder                                             │
│      입력: 전체 JSON 취합 + 이미지                          │
│      출력: outputs/*.hwpx                                 │
│      에이전트: ngd-exam-builder (기존)                      │
└───────────────────────┬─────────────────────────────────┘
                        ▼
┌─────────────────────────────────────────────────────────┐
│  [6] Checker                                             │
│      입력: 생성된 HWPX                                    │
│      출력: pass/fail + 수정 지시                            │
│      에이전트: ngd-exam-checker (기존)                      │
└─────────────────────────────────────────────────────────┘
```

## V1 → V2 → V3 비교

| 항목 | V1 | V2 | V3 |
|------|-----|-----|-----|
| 입력 방식 | PDF 1장 | PDF + 문제 이미지 | 문제 이미지 (수동 크롭) |
| 텍스트 추출 | PDF → 텍스트 (전체) | PDF + 이미지 2차검증 | 이미지 → 문제별 개별 추출 |
| 처리 단위 | 시험지 전체 | 시험지 전체 | **문제 1개** |
| 병렬 처리 | 없음 | 없음 | **4개 동시** |
| 해설 생성 | solver 1회 일괄 | solver 1회 일괄 | **문제별 독립 생성** |
| 해설 검증 | checker에서 간접 | checker에서 간접 | **verifier 전용 에이전트** |
| 교과 순서 | 미반영 | 미반영 | **단원 컨텍스트 제공** |
| 재시도 | checker 2회 | checker 2회 | **verifier 3회 + checker** |
| 그림 처리 | figure 에이전트 | figure 에이전트 | figure 에이전트 (동일) |

## 신규 에이전트

| 에이전트 | 역할 | 입력 | 출력 |
|---------|------|------|------|
| `ngd-exam-extractor` | 이미지에서 문제 추출 | 문제 이미지 1장 | 문제 JSON |
| `ngd-exam-verifier` | 해설 검증 | 이미지 + JSON | pass/fail + 피드백 |

## 기존 에이전트 변경

| 에이전트 | 변경 내용 |
|---------|----------|
| `ngd-exam-solver` | 문제별 독립 호출, 교과 순서 컨텍스트 추가, 난이도별 줄 수 제한 제거 |
| `ngd-exam-figure` | 변경 없음 (기존 유지) |
| `ngd-exam-builder` | 변경 없음 (JSON 포맷 동일) |
| `ngd-exam-checker` | 변경 없음 (기존 유지) |

## 기존 에이전트 폐기 (V3에서 미사용)

| 에이전트 | 이유 |
|---------|------|
| `ngd-exam-reader` | extractor가 대체. 이미지 기반이므로 PDF 텍스트 추출 불필요 |
