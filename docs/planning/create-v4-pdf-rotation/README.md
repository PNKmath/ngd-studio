# create-v4 PDF 회전 지원

## 배경

`/create-v4`에서 PDF를 불러올 때 일부 시험지가 180도 뒤집힌 상태로 렌더링되는 경우가 있다. 현재 cropper는 렌더된 PNG 픽셀 좌표계를 기준으로 박스 생성, 자동 분할 결과 주입, 최종 PNG 추출을 모두 처리한다. 따라서 화면만 CSS로 회전하면 박스 좌표와 실제 추출 결과가 어긋난다.

## 목표

PDF preview/meta/auto-crop/extract 흐름이 동일한 회전값을 공유하도록 만들어, 사용자가 뒤집힌 PDF를 회전한 뒤 수동 박스 조정과 자동 분할을 같은 기준에서 수행할 수 있게 한다.

## 성공 기준

- `/create-v4`와 `/pdf-cropper`에서 업로드한 PDF를 90도 단위로 회전할 수 있다.
- 회전된 미리보기 위에서 박스를 만들고 조정한 뒤 추출한 PNG가 화면에 보이는 영역과 일치한다.
- 자동 분할 실행 시 Gemini 입력 이미지와 반환 bbox가 현재 회전 기준과 일치한다.
- 기존 비회전 PDF 흐름과 `/pdf-cropper` ZIP 다운로드 흐름이 회귀하지 않는다.

## 제약

- 회전 범위는 우선 전체 PDF 공통 회전으로 제한한다. 페이지별 회전은 별도 후속 작업으로 둔다.
- 좌표계는 기존처럼 "렌더된 PNG 픽셀 좌표계"를 유지한다.
- 단순 CSS transform 기반 회전은 사용하지 않는다.
