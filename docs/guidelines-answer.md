# NGD 해설/정답/서술형 규칙

양식지에서 추출한 해설, 정답, 서술형 관련 규칙.

## 서술형 표기

- 항상 **`[서술형 N]`** 형식으로 입력 (N은 일반 숫자)
- `[단답형]`, `[논술형]`으로 되어 있어도 모두 `[서술형 N]`으로 변환

## 서술형 소문항

- 서술형에 딸린 소문항 번호 `1), 2)`, `i), ii)`, `(1), (2)` 등은 모두 **`(1), (2)`로 통일**
- 원본 문제가 `(1)` 형식이 아니더라도 `(1)`로 변형
- 일반 타이핑 (특수기호/수식 아님)

## 서술형 배점

- 소문항에 점수가 있으면 **전체 배점은 적지 않음**
- 소문항에만 배점을 기입
- 소문항 배점이 없으면 전체 배점 기입 (`paraPrIDRef="4"` RIGHT 정렬)

## 해설 작성

- 정답 해설은 학생들이 익힐 수 있도록 **상세히 풀이** 작성
- **쎈 교재 수준**으로 작성하는 것을 기본으로 함
- `[다른 풀이]`가 있으면 `[다른 풀이]`라 입력하고 추가 작성

## 정답 라인

- 해설 구조: **[정답] 라인 → 풀이** 순서
- 정답에 **bold 처리 금지** (수식도 bold 금지)
- 16페이지의 기본 `[정답]` 틀은 진하기/밑줄 등 수정하지 않음
- 정답 라인 밑에 **"해설", "풀이"라는 문구를 넣지 않음**

## shift+enter 규칙

- **shift+enter(`<hp:lineBreak>`) 사용 금지**
- 유일한 예외: `[정답]` 라인이 길어져서 **2줄로 넘어갈 때만** 허용
- 이유: 빠른정답 생성 시 정답 전체를 긁어올 수 있도록

## 해설 수식 규칙

- **통수식 금지** — 등호 단위로 끊어서 입력
- 경우를 나누어 푸는 경우: **ⅰ), ⅱ)** 또는 **(ⅰ), (ⅱ)** — 문자표 로마자 사용

## 연방풀이 (연립방정식)

- 연방풀이에서 원문자 번호는 **(1), (2)가 아닌** 문자표의 **㉠, ㉡** 사용
- 예시:
  ```
  cases{2x^2 - xy - 12y^2 = 3 & ````` cdots CDOTS ````㉠
        # x^2 - 4xy + y^2 = -2 & `` ``` cdots CDOTS ````㉡}
  ```

## cdotscdots 뒤 원문자

- `cdotscdots` 다음의 원문자(㉠, ㉡ 등)는 **수식 내부가 아닌 바탕글(본문)에서** 작성
