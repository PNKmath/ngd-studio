# NGD 단서 조항 처리 규칙

문제 뒤의 단서 조항(예: "단, a는 양수이다.")과 배점 처리 규칙.

## 기본 원칙

문제 텍스트 뒤에 단서 조항 + 배점이 모두 들어가면 **그대로 작업**한다.

## 한 줄에 안 들어가는 경우

### Case 1: 단서 조항이 한 줄
- 엔터 치고 **오른쪽 정렬** (`paraPrIDRef="4"`)
- 예시:
  ```
  좌표평면 위의 네 점 A(-2, -1), B(-3, a), C(b, c), D(2, 0)에 대하여
  사각형 ABCD가 마름모를 이룰 때, a+b+c의 값은?
                                    (단, a, b, c는 양수이다.) [4.4점]
  ```

### Case 2: 단서 조항이 두 줄 이상
- 문제 뒤에 **이어서 작업**한다 (엔터 안 침)
- 예시 (확률/통계 문제에서 표준정규분포 단서가 긴 경우):
  ```
  정규분포 N(m, 10²)을 따르는 모집단에서 크기가 n인 표본을 임의추출할 때,
  표본평균과 모평균의 차가 3α+1 이하일 확률은 0.9974이다. 이때 자연수 n, α에
  대하여 n+α의 값은? (단, Z가 표준정규분포를 따르는 확률변수일 때,
  P(0≤Z≤3)=0.4987로 계산하다.) [4.3점]
  ```

## 배점만 남는 경우

- 단서 조항을 이어서 쓴 후에도 **배점만 한 줄에 둥그러니** 남으면:
  - 배점만 따로 **오른쪽 정렬** (`paraPrIDRef="4"`)

## XML 구현

- 오른쪽 정렬이 필요한 줄: `paraPrIDRef="4"` (RIGHT 정렬 문단)
- 이어서 작업하는 경우: 같은 `<hp:p>` 문단 안에 계속 작성
