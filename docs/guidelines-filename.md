# NGD 파일명/머리말/단원 규칙

## 파일명 구조

PDF 제목:
```
[코드][고][년도][학기-차수][지역][학교][과목][코드]
```
예: `[01001][고][2019][1-2-a][충남공주시][공주사대부고][수하][01001]`

작업 HWPX 제목:
```
[코드][고][년도][학기-차수][지역][학교][과목][범위][코드][작업자][검수자][그림코드]
```
예: `[01001][고][2019][1-2-a][충남공주시][공주사대부고][수하][도형의 이동-명제][01001][그림2-3-1-0]`

## 그림 코드

`[그림A-B-C-D]` 형식:
- A: 문제 그림 총 개수
- B: 해설 그림 총 개수
- C: 작업자가 그린 문제 그림 수
- D: 작업자가 그린 해설 그림 수

예: `[그림2-3-1-0]` = 문제그림 2개 중 1개 작업, 해설그림 3개 중 0개 작업

## 범위 작성 (파일명 vs 문서 내)

### 파일명의 범위
- **대단원명까지만** 기재
- 도함수활용 등 세부 단원은 파일명에 포함하지 않음
- 예: `[함수의극한-도함수의활용-1]` (O), `[함수의극한-접선-평균값정리]` (X)

### 문서 내 [중단원] 태그
- 양식지 단원분류표의 **정규 단원명을 그대로** 사용
- 예: `도함수활용-1 접선-평균값정리(미적)` — 전체 다 기재

## 단원 분류 주의사항

- 수II와 미적분의 도함수활용 단원은 **4개로 세분화**:
  - `도함수활용-1 접선-평균값정리(수II)`
  - `도함수활용-2 극대극소-최대최소(수II)`
  - `도함수활용-3 방정식-부등식(수II)`
  - `도함수활용-4 변화율-속도-가속도(수II)`
- 반드시 `.claude/data/unit_classification.json` 참조

## 난이도

4단계: **하 / 중 / 상 / 킬**
- "최상", "최하", "보통" 등 비표준 난이도 금지

## 머리말 정보

양식의 머리말 오른쪽 칸에 기재:
- **학년/과정/과목/출판사** + **시험 범위**
- 고1: 학년/과목/출판사 + 범위
- 출판사 모를 때는 생략

## PDF 단원명 오류

- PDF에 단원명이 잘못되어 있으면 **수정해서 제출**
